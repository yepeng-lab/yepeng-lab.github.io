Linux入门——shell基础

发表于 2020-04-19 | 分类于 Linux |

Shell概述

什么是shell

命令解释器，交互界面

将命令转化为机器语言，将机器语言翻译成自然语言

Windows的shell：图像，侦测到鼠标单击，就触发shell

shell也是一个功能强大的编程语言，易编写、易调试、灵活性强。shell是解释执行的脚本语言，在shell中可以直接调用Linux系统命令

shell的分类

  • Bourne Shell

  • C Shell：语法结构类似c语言

  • Linux主流Shell：Bash

Linux支持的shell

/etc/shells

shell脚本的执行方式

echo输出命令

echo [选项][输出内容]

选项：

-e：支持反斜线控制的字符转换

比如输出颜色：echo -e “\e[1;31m abcd \e[0m”

#30m 黑色， 31m红色，32m绿色，33m黄色
#34m蓝色， 35m洋红，36m青色，37m白色

Bash基本功能

历史命令与补全

历史命令

history [选项] [历史命令保存文件文件]

-c 清空历史命令

-w 把缓存中的历史命令写入历史命令保存文件 ~/.bash_history

  1. 历史命令默认会保存1000条，可以在环境变量配置文件按/etc/profile中进行修改

  2. 历史命令的调用

    • 上下箭头

    • !n 重复执行第n条命令

    • !! 重复执行上一条命令

    • “!string“执行最后一条以string开头的命令

命令补全

  1. Tab键补全

命令别名与快捷键

alias command= ‘commandName‘

vi /root/.bashrc

unalias 别名

命令执行顺序

  1. 绝对路径或相对路径

  2. 第二顺位执行别名

  3. 第三顺位执行Bash的内部命令

  4. 第四顺位执行按照$PATH环境变量定义的目录查找顺序找到第一个命令

快捷键

常用快捷键

快捷键 作用
ctrl+c 强制停止
Ctrl+L 清屏
Ctrl+U 删除(剪切)光标前的内容
Ctrl+y 粘贴
Ctrl+r 在历史命令中搜索
Ctrl+d 退出当前终端

输入输出重定向

标准输入输出

设备 设备文件名 文件描述符 类型
键盘 /dev/stdin 0 标准输入
显示器 /dev/stdout 1 标准输出
显示器 /dev/stderr 2 标准错误输出

输出重定向

改变输出方向，改屏幕为文件，大多是给管理员随时查看

类型 符号 作用
标准输出重定向 命令>文件 已覆盖的方式
标准输出重定向 命令>>文件 以追加的方式
标准错误输出重定向 命令2>文件 以覆盖的方式
标准错误输出重定向 命令2>>文件 以追加的方式
更实用的输出重定向

命令 >　文件 2>&1

命令 >> file 2>&1

commmand &> file

commmand &>> file

command >> file2 2>> file2

wc 输入重定向

wc 选项 文件名

选项

  • -c 统计字节数

  • -w 统计单词数

  • -l 统计行数

命令 <　文件

多命令顺序执行与管道符

多命令顺序执行

规定m=||

执行符 格式 作用
； 命令1；命令2 命令顺序执行，没有逻辑联系
&& 命令1&&命令2 只有命令1正确执行，才执行命令2
m｜命令1m命令2 只有命令1错误执行才执行命令2
  • dd if=输入文件 of=输出文件 bs=字节数 count=个数

主要用来进行磁盘复制

例如： date; dd if=/dev/zero of=/root/testfile bs=1k count 1000000;date

管道符

命令1的正确输出作为命令2的操作对象

比如： ll -a /etc/ | more

注意：命令1必须正确执行

grep 选项 “搜索内容” 文件名

选项

  • -i 忽略大小写

  • -n 输出行号

  • -v 反向查找

  • –color=auto 用颜色显示关键字

通配符与

通配符（匹配文件名）

  1. ? 匹配任意一个字符

  2. *匹配任何内容

  3. []陪陪括号中任意一个字符

  4. [-]匹配括号中范围任意一个

  5. [^]逻辑非，匹配不属于范围的

其他符号

  • ‘’ 单引号，所有特殊符号都没有含义

  • “” 双引号，特殊符号有特殊意义

  • `` 反引号，系统命令，首先执行，与$()一样

  • $ 与反引号一样

  • #注释

  • $ 用于调用变量的值

  • \ 转义符，跟在\之后的特殊符号将失去特殊含义

Bash变量

用户自定义变量

Linux默认类型是字符串

变量赋值等号两侧不能有空格

变量值有空格加引号

变量可以叠加

命令的结果可以赋给变量，要用``和$()符号括起来

环境变量建议大写

变量类型

  1. 用户自定义变量

  2. 环境变量：和操作系统环境相关的数据

  3. 位置参数变量：向脚本传递参数和数据，不能自定义，只能改值

  4. 预定义变量：类似位置参数变量

本地变量（用户自定义变量）

x=value

用echo调用

变量叠加，用${x}value或者x="$x"value

变量查看：set

变量删除unset

环境变量

环境变量是什么

用户自定义变量只在当前shell生效，环境变量会在当前Shell和它所有子shell生效，如果写入配置文件，那么就会在所有shell生效

设置环境变量

export 变量名=变量值

env 查询变量

unset

两个环境变量

PATH

系统查找命令的路径，用:连接

比如：PATH=”$PATH”:/root

PS1

提示符

位置参数变量

位置参数变量 作用
$n n为数字，$0代表命令本身，$1-$9代表第一到第九歌参数，大于10用{}连接
$* 返回命令行所有的参数，看成整体
$@ 返回所有参数，区别对待
$# 返回参数个数

预定义变量

  • $? 返回最后一次执行命令的返回状态：0或非0

  • $$ 显示当前进程的进程号

  • $! 显示后台最后一个进程的进程号

接受键盘输入

read 选项 变量名

选项：

  • -P “提示信息” ：等待read输入时，输出提示信息

  • -n 秒数 等待时间

  • -n 字符数，接收指定字符数后结束

  • -s 隐藏输入

数值运算与运算符

declare声明变量类型

declare [+/-]选项 变量名

  • 变量设置属性

数值运算方法
