Linux入门――软件管理包

发表于 2020-04-17 | 分类于 Linux |

简介

软件包分类

源码包(绝大多数是C语言，部分C++)

- 脚本安装包

- 优点

    - 开源，可以修改源代码

    - 可以自由选择功能

    - 软件是编译安装，所以更适合自己的系统，更加稳定也效率更高

    - 卸载更加方便(直接删目录)

- 缺点

    - 安装步骤较多，尤其是安装较大的软件集合时，容易出现拼写错误

    - 编译时间过长，安装比二进制安装时间长

    - 因为是编译安装，安装过程中一旦报错新手很难解决

二进制包(经过编译之后)

- RPM包(CentOS)

- 系统默认包

- 优点

    - 包管理系统简单，只需要几个简单的命令就可以实现安装、升级、查询、搜索

    - 安装时间长

- 缺点

    - 不能看见源代码

    - 功能选择不如源码包灵活

    - 依赖性

RPM包管理——RPM命令管理

包命名与依赖性

命名规则：如httpd-2.2.15-15.e16.centos.1.i686.rpm()

名 意义
httpd 软件包名
2.2.15 软件版本
15 软件发布的次数
e16.centos 适合的Linux平台
i686 适合的硬件平台
rpm rpm包扩展名

依赖性

  • 树形依赖：a->b->c

  • 环形依赖：a->b->c->a

  • 模块依赖

rpm命令管理——安装升级与卸载

包命名与包名

  1. 包全名：操作没有安装的软件包

  2. 包名：操作已经安装的软件包，实际上是搜索/var/lib/rpm/下的数据库，比如查询、卸载

安装

  • rpm -ivh 包全名

选项：

- -i(install) 安装

- -v(verbose) 显示详细信息

- -h(hash) 显示进度

- --nodeps    不检测依赖性

升级

  • rpm -Uvh 包全名

卸载

rpm -e 包名

选项：
- -e(erase) 卸载
- –nodeps 不检查依赖性

rpm命令管理——查询

查询软件包是否安装

  • rpm -q 包名

  • rpm -qa 包名

查询软件包详细信息

rpm -qi 包名

选项：

- -i 查询软件信息(information)

- -p 查询未安装包信息(package)

rpm -ql 包名

- -l 列表(list)

- -p 查询未安装包信息(package)

查询系统文件属于哪个RPM包

rpm -qf 系统文件名

选项：

- -f(查询系统文件属于哪个文件名)

查询软件包的依赖性

rpm -qR 包名

选项：

- -R 查询软件的依赖性(requires)

- -p 查询未安装包信息(packages)

rpm校验包校验

RPM包校验

rpm -V 已安装的包名

选项：

- -V 校验指定RPM包中的文件(verify)

注意：信息默认8个字符，具体信息如下

字母 信息
S 文件大小
M 文件的类型和文件的权限是否改变
5 文件MD5校验码是否改变
T 文件的修改时间是否改变

文件类型

|c|配置文件|
|d|普通文档|
|g|鬼文件(ghost file)，意思是该文件不应该被|

RPM包中提取文件

rpm2cpio #rpm包转换为cpio的命令

cpio #一个标准工具，它用于创建软件文档文件和从文档文件中提取文件

yum在线管理

为了解决rpm依赖性，便出现了yum在线管理，只要你需要什么软件包，yum就会自动下载这个软件包和它依赖的软件包，但是有可能让你给钱

IP地址配置和网络yum源

配置ip

使用setup工具，然后配置文件，执行命令:vim /etc/sysconfig/network-scripts/ifcfg-eth0

将ONBOOT="no"改为ONBOOT="yes" #启动网卡

执行service network restart

网络yum源：保存在/etc/yum.repos.d/ ,默认是CentOS-Base.repo

  • [base] 容器名称，一定要放在[]里面

  • name 容器说明，可以自己随便写

  • mirrorlist 镜像站点，这个可以注释掉

  • baseurl yum源服务器的地址。默认是CentOS官方的源服务器，是可以使用的

  • enable 默认是0

  • gpgcheck 如果是1是指RPM的数字证书生效，0则是不生效

  • gpgkey 数字证书的公钥保存位置。不用修改

yum命令

查询

  1. yum list：查询所有可用的软件包列表

  2. yum search 关键字(包名)：搜索服务器上所有和关键字相关的包

安装

  1. yum -y install 包名

选项
- install：安装
- -y：自动回答yes

  1. yum -y update 包名

选项：

- update：升级

- -y 自动回答yes

卸载

  1. yum -y remove 包名

YUM软件包组管理命令

  1. yum grouplist：查询所有软件组列表

  2. yum groupinstall 软件组名：安装指定软件组，组名可以用上一条命令查询出来

  3. yum groupremove 软件组名：卸载指定软件组

yum光盘源搭建

  1. 挂载光盘

    mount 硬件设备 挂载点

  2. 让网络yum源文件失效

    改变后缀名

  3. 修改yum源文件

    [c6-media]
    name=CentOS-$releasever - Media
    baseurl=file:///mnt/cdrom #光盘挂载地址

    file:///media/cdrom/

    file:///media/cdrecorder/

    #注释这两个不存在的地址

源码包管理

源码包与RPM包的区别

  1. 安装之前的区别：概念上的区别

  2. 安装之后的区别：安装位置的区别

RPM包安装位置

  1. 在默认位置中

|RPM包默认安装路径|
|——|———|
|/ect/|配置文件安装目录|
|/usr/bin/|可执行的命令安装目录|
|/usr/lib/|程序所使用的函数库保存位置|
|/usr/share/doc/|基本软件使用手册保存位置|
|/usr/share/man/|帮助文件保存位置|

源码包安装位置

  1. 安装在指定位置，一般是/usr/local/软件名/

安装位置不同带来的影响

  1. rpm包安装的服务可以使用系统服务管理命令(service)来管理，例如RPM包安装的apache的启动方式是：

    /etc/rc.d/init.d/httpd start
    systemctl httpd start

  2. 源码包安装的服务则不能被服务管理命令管理，因为没有安装到默认路径中。所以只能用绝对路径进行服务的管理，如：
    /usr/local/apache2/bin/apacecectl start

源码包安装过程

安装准备

  1. 安装C语言编译器

  2. 下载源码包：http://mirror.bit.edu.cn/apache/httpd/

安装注意事项

  1. 源码包的保存位置：/usr/local/src/

  2. 软件安装位置：/usr/local/

  3. 如何确定安装过程报错：

    • 安装过程停止

    • 出现error/waring/no的提示

安装过程

  1. 下载源码包

  2. 解压缩下载的源码包

  3. 进入解压缩目录

  4. 查看ISTALL和README

  5. ./configure 软件配置与检查

    • 定义所需要的功能选项

    • 检查系统环境是否符合安装要求

    • 把定义好的功能选项和检测系统环境的信息都写入Makefile文件,用于后续的编辑
